Issues

Interestingly, I had an easy time setting up the server I wrote with only minor issues. For multiple clients I use a multiplexing I/O system with the select() function. select() tracks file descriptors and alerts the OS of any new activity. Right now, it is only active for up to 10 clients at a time. This is because the Pi has 1GB of RAM and im worried about it using too much memory.

Unsupported Methods

This server only supports GET and HEAD requests. This is because I'm not sure how to handle POST or PUT requests yet. OPTIONS i may add as well. Not sure how or when.

Favicon

For those of you that don't know, favicons are those tiny images that appear in the upper left of the tab. Yes, your browser makes a seperate request for each one of those. When I say these are a pain, they are. They only support a few different filetypes. I use svg as a filetype for mine although I have no idea if that's usual. I look at the Accept: header from my browser and it asks for it. It's not so much the complexity of the favicon request, it's the sparcity of the requests. Your browser will cache the webpage but if you refresh the page, it'll send a request for the page. Guess what it won't send a request for? Thats right, the favicon. Good luck debugging further on that future me. :(
